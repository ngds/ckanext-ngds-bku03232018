We Weren't Able to Process Your OGC Service

Here's a list of the most common errors users run into:

  • OGC Compliance
    This system only supports OGC (Open Geospatial Consortium) WMS and WFS services. If this service has been uploaded through NGDS from an ESRI Shapefile or a CSV file, then this is an OGC service served through Geoserver. If this is a linked data service, it is possible that it does not conform to OGC specifications.

  • Service Endpoint Confusion
    ArcGIS Server distinguishes between an ArcGIS REST Service and an OGC Service. A service endpoint for an ArcGIS REST Service might look something like this: http://services.azgs.az.gov/arcgis/rest/services/aasggeothermal/TXActiveFaults/MapServer and a service endpoint for an ArcGIS OGC WFS Service might look something like this: http://services.azgs.az.gov/arcgis/services/aasggeothermal/TXActiveFaults/MapServer/WFSServer?.

  • Layer(s) Nonexistent
    When uploading data, users should specify an appropriate layer name in the OGC service. There are certain use cases where a user might not have the option to specify a layer name. In these cases, the system is designed to select the first layer name in the list of layer names in the service. If there are no layers or layer names, this will result in an invalid request and no data will be returned.

  • Input Error
    Oftentimes, the culprit is a simple error like the misspelling of a URL or a typo in another input dialogue.
